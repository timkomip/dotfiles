# review-project

## Overview

Reviews a project and provides a comprehensive report on code health, quality, and security.

## Review Categories

### Functionality
- [ ] Code implements intended functionality correctly
- [ ] Edge cases are handled appropriately
- [ ] Error handling is present and appropriate
- [ ] No obvious bugs or logic errors

### Code Quality
- [ ] Code is readable and well-structured
- [ ] Functions are appropriately sized and focused (avoid over-engineering)
- [ ] Variable and function names are descriptive and clear
- [ ] Minimal code duplication (DRY principle)
- [ ] Follows project conventions and style guidelines
- [ ] No typos or grammatical errors in code/comments
- [ ] Code flow is easy to follow for humans
- [ ] Configuration is easy to extend or modify

### Security
- [ ] No obvious security vulnerabilities
- [ ] Input validation is present where appropriate (pragmatic approach)
- [ ] Sensitive data is handled securely
- [ ] No hardcoded secrets or credentials
